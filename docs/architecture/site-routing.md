# Site routing

Current as of 2026-06-30.

## Summary

The current blog route surface reads from the Astro `posts` collection. Blog route pages do not query Notion repositories directly.

The current friends route surface reads active Friends through the friends Notion repository.

[src/content.config.ts](../../src/content.config.ts) defines the `posts` collection and uses [src/server/notion/index.ts](../../src/server/notion/index.ts) as the app-facing Notion loader entry. The collection schema is documented in [Content model](./content-model.md).

## Blog route surfaces

The current blog route surfaces are:

- `/posts`
- `/posts/[slug]`
- `/categories`
- `/categories/[category]`
- `/tags`
- `/tags/[tag]`

[src/pages/posts/index.astro](../../src/pages/posts/index.astro) lists posts from `getCollection("posts")` and sorts them by `publishedDate` descending.

[src/pages/posts/[slug].astro](../../src/pages/posts/%5Bslug%5D.astro) builds static paths from `post.id`, passes each `CollectionEntry<"posts">` as route props, and renders the entry with Astro `render(post)`.

[src/pages/categories/index.astro](../../src/pages/categories/index.astro) derives the category index from non-empty `post.data.category` values in the `posts` collection.

[src/pages/categories/[category].astro](../../src/pages/categories/%5Bcategory%5D.astro) builds static category paths from collection categories and passes the decoded category label through route props.

[src/pages/tags/index.astro](../../src/pages/tags/index.astro) derives the tag index from `post.data.tags` values in the `posts` collection.

[src/pages/tags/[tag].astro](../../src/pages/tags/%5Btag%5D.astro) builds static tag paths from collection tags and passes the decoded tag label through route props.

## Friends route surface

The current friends route surface is:

- `/friends`

[src/pages/friends/index.astro](../../src/pages/friends/index.astro) loads the friends page state and view model through [src/lib/friends-page.ts](../../src/lib/friends-page.ts). The page state calls [src/server/notion/friends/repository.ts](../../src/server/notion/friends/repository.ts). The view model carries the page heading, notice copy, response status, and Friend cards used by the route.

[src/config.ts](../../src/config.ts) currently enables the friends route through `SITE_FEATURES.friends.route`. [src/components/NavBar.astro](../../src/components/NavBar.astro) uses the same setting to show the Friends navigation link. When the friends route is disabled in site config, [src/pages/friends/index.astro](../../src/pages/friends/index.astro) sets a 404 response status and renders the disabled state.

## Route parameter encoding

Category and tag links are generated by [src/lib/content-routes.ts](../../src/lib/content-routes.ts).

`termRouteParam` encodes a label with `encodeURIComponent`, escapes additional path-sensitive characters, replaces percent signs with `~`, and prefixes the result with `t_`.

`categoryHref` and `tagHref` produce trailing-slash route URLs under `/categories/` and `/tags/`.

The dynamic category and tag route pages do not decode the route parameter. They generate static paths from collection values and carry the original category or tag label in route props.

## Current constraints

The `posts` collection snapshot is the current publishing exit for blog route pages.

The `/friends` route is outside the Astro blog content sync and uses the friends repository directly. The current site config keeps the friends route enabled.
